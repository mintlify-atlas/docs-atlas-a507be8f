Component tokens are CSS custom properties scoped to a specific UI component. They sit at the top of Primer's three-layer token hierarchy, consuming functional tokens (like `--bgColor-*` and `--fgColor-*`) and adapting to all themes automatically.

<Note>
Component tokens are intended for use **inside the component they belong to**. Do not use `--button-*` tokens to style arbitrary elements â€” use functional tokens instead.
</Note>

---

## How component tokens work

Component tokens reference functional tokens rather than raw values:

```css
/* Component token references a functional token */
--button-default-bgColor-rest: var(--control-bgColor-rest);

/* Functional token resolves to the correct value per theme */
--control-bgColor-rest: #f6f8fa; /* light mode */
--control-bgColor-rest: #21262d; /* dark mode */
```

This means you get automatic theme support without writing any theme overrides yourself.

## Button tokens

The button component has four variants: `default`, `primary`, `invisible`, `outline`, and `danger`. Each variant exposes color tokens for foreground, background, border, and shadow across all interactive states.

|---|---|---|---|
| `--button-default-fgColor-rest` | color | rest | Label color at rest |

## Overlay tokens

Overlay tokens apply to modal dialogs, popovers, and drawer panels.

| Token | Type | Description |
|---|---|---|
| `--overlay-bgColor` | color | Background color of the overlay surface |
| `--overlay-borderColor` | color | Border color of the overlay; semi-transparent by default |
| `--overlay-backdrop-bgColor` | color | Background color of the backdrop scrim behind the overlay |

```css
.dialog {
  background-color: var(--overlay-bgColor);
  border: var(--borderWidth-thin) solid var(--overlay-borderColor);
}

.backdrop {
  background-color: var(--overlay-backdrop-bgColor);
}
```
