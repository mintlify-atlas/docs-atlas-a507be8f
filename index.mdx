Primer Primitives is the token layer powering GitHub's Primer design system. It provides a comprehensive set of design tokens compiled by Style Dictionary into CSS custom properties, JSON, and Figma variables — covering color themes, typography scales, spacing, motion, borders, and shadows.

## Get started in three steps

<Steps>
  <Step title="Install the package">
    Add `@primer/primitives` to your project via npm, yarn, or pnpm.

    ```bash
    npm install @primer/primitives
    ```
  </Step>
  <Step title="Import a theme">
    Import a color theme CSS file to set up CSS custom properties for your chosen color mode.

    ```css
    @import '@primer/primitives/dist/css/functional/themes/light.css';
    @import '@primer/primitives/dist/css/base/typography/typography.css';
    @import '@primer/primitives/dist/css/functional/size/size.css';
    ```
  </Step>
  <Step title="Use CSS variables in your styles">
    Reference tokens as CSS custom properties in your stylesheets or components.

    ```css
    .card {
      background-color: var(--bgColor-default);
      color: var(--fgColor-default);
      border: 1px solid var(--borderColor-default);
      border-radius: var(--borderRadius-medium);
      padding: var(--stack-padding-normal);
    }
    ```
  </Step>
</Steps>

## Why Primer Primitives?

<CardGroup cols={2}>
  <Card title="Multi-theme support" icon="swatchbook">
    Nine built-in themes including light, dark, dark dimmed, high-contrast variants, and colorblind-friendly modes.
  </Card>
  <Card title="Accessibility first" icon="universal-access">
    All color pairings are validated against WCAG AA contrast requirements, with automated contrast checking built into CI.
  </Card>
  <Card title="Figma integration" icon="figma">
    Tokens compile directly to Figma variables, keeping design and code in sync without manual handoff.
  </Card>
  <Card title="Style Dictionary powered" icon="gear">
    Built on the industry-standard Style Dictionary, with custom transformers, formats, and preprocessors for Primer's needs.
  </Card>
</CardGroup>

<Note>
